



# ML — Evaluation

---

## 1. Classical ML

### Classification Metrics

| Metric    | Formula                     | Note                                   |
| --------- | --------------------------- | -------------------------------------- |
| Accuracy  | `correct / total`           | Misleading on imbalanced data          |
| Precision | `TP / (TP + FP)`            | Of predicted positives, how many real? |
| Recall    | `TP / (TP + FN)`            | Of real positives, how many caught?    |
| F1        | `2 · (P·R) / (P+R)`         | Balance of precision/recall            |
| AUC-ROC   | Area under TPR vs FPR curve | Threshold-independent                  |
| Log Loss  | `-Σ y·log(ŷ)`               | Penalizes confident wrong predictions  |
| ...       |                             |                                        |

**Confusion matrix** is the foundation — always look at it before summarizing with one number.

### Regression Metrics

| Metric | Formula             | Note                           |
| ------ | ------------------- | ------------------------------ |
| MSE    | `(1/n)Σ(y-ŷ)²`      | Penalizes large errors more    |
| RMSE   | `√MSE`              | Same units as target           |
| MAE    | `(1/n)Σ\|y-ŷ\|`     | Robust to outliers             |
| R²     | `1 - SS_res/SS_tot` | 1 = perfect, 0 = predicts mean |
| ...    |                     |                                |

### Clustering Metrics

| Metric                       | Note                                                           |
| ---------------------------- | -------------------------------------------------------------- |
| Silhouette score             | How similar a point is to its cluster vs others. Range [-1, 1] |
| Inertia (within-cluster SSE) | Lower = tighter clusters; use for elbow method                 |
| Adjusted Rand Index          | Needs ground truth labels                                      |
| ...                          |                                                                |

## 2. Validation Strategy

|Strategy|How|Use|
|---|---|---|
|Hold-out split|Train / Val / Test fixed split|Large datasets|
|k-Fold CV|Split into k folds, rotate val set|Small datasets|
|Stratified k-Fold|k-Fold preserving class balance|Imbalanced classification|
|Time-series split|Train on past, val on future only|Sequential data — no leakage|

**Golden rule:** test set is touched once, at the very end.

## 3. Neural Networks

### During Training

|Signal|What to watch|
|---|---|
|Train loss|Should decrease steadily|
|Val loss|Should track train; divergence = overfitting|
|Train/val gap|Large gap = overfitting; both high = underfitting|
|Gradient norms|Sudden spikes = exploding gradients|

### Image (CNN)

Same classification/regression metrics as above, plus:

|Metric|Use|
|---|---|
|Top-5 accuracy|ImageNet-style; correct if true label in top 5|
|mAP (mean avg precision)|Object detection|
|IoU|`Intersection / Union` — bounding box or segmentation overlap|

### Language Models

|Metric|Formula / Note|
|---|---|
|Perplexity|`exp(NLL)` — lower = better; measures surprise|
|BLEU|N-gram overlap with reference (translation, summarization)|
|ROUGE|Recall-oriented n-gram overlap (summarization)|
|BERTScore|Semantic similarity via embeddings|
|Accuracy / F1|For classification heads (NLU tasks)|

### Generative Models

|Metric|Use|
|---|---|
|FID (Fréchet Inception Distance)|Image generation quality|
|IS (Inception Score)|Quality + diversity of generated images|
|ELBO|VAE training objective (reconstruction + KL)|

## 4. Common Mistakes

|Mistake|Why it's wrong|
|---|---|
|Evaluating on training data|Measures memorization not generalization|
|Accuracy on imbalanced data|Misleads — 99% acc on 99/1 split = trivial model|
|Data leakage into val/test|Preprocessing fitted on full dataset before split|
|Single metric only|Always look at multiple + confusion matrix|
|No error analysis|Knowing where the model fails is more useful than the number|
