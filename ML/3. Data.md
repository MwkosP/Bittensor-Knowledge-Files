


# ML — Data


## DATA QUALITY AND PREPROCESSING IS ONE OF THE MOST IMPORTANT THINGS IN ML.  
---

## 1. Data Splits

| Split      | Purpose                | Typical Size |
| ---------- | ---------------------- | ------------ |
| Train      | Model learns from this | 70–80%       |
| Validation | Tune hyperparameters   | 10–15%       |
| Test       | Final evaluation, once | 10–15%       |

**Split before any preprocessing.** Fit transformers on train only, apply to val/test.
Percentages and splits can vary.

---

## 2. Preprocessing

### Numerical

|Step|Why|
|---|---|
|Standardization `(x-μ)/σ`|Zero mean, unit variance — needed for gradient-based models|
|Min-Max scaling `(x-min)/(max-min)`|Bounds to [0,1] — good for bounded inputs|
|Log transform `log(x+1)`|Compresses skewed / long-tail distributions|
|Clipping|Remove extreme outliers before scaling|

### Categorical

|Method|Note|
|---|---|
|One-hot encoding|For low cardinality (`< ~20` categories)|
|Label encoding|Ordinal categories only|
|Embedding|High cardinality — learned during training|
|Target encoding|Replace category with mean target — risk of leakage|

### Missing Values

| Strategy                          | When                                                      |
| --------------------------------- | --------------------------------------------------------- |
| Drop rows                         | Few missing, large dataset                                |
| Mean / median imputation          | Numerical, not too many missing                           |
| Mode imputation                   | Categorical                                               |
| Model imputation (KNN, iterative) | High missingness, important feature                       |
| Indicator column                  | Add `feature_missing` binary flag alongside imputed value |

## 3. Feature Engineering

|Technique|Example|
|---|---|
|Polynomial features|`x₁², x₁·x₂`|
|Binning|Age → age group|
|Date decomposition|Timestamp → hour, day, weekday, month|
|Interaction terms|Combine two features manually|
|Aggregations|Group-by statistics (mean, std per user)|
|Dimensionality reduction|PCA, UMAP — for high-dim tabular|

Feature selection:

- Correlation with target
- Feature importance (tree models)
- Variance threshold (drop near-zero variance)
- Recursive feature elimination (RFE)

## 4. Tokenization (Text)

Converts raw text → integer token IDs for model input.

|Method|Idea|Used In|
|---|---|---|
|Word-level|Split on whitespace|Old NLP|
|Character-level|One token per character|Rare|
|BPE (Byte-Pair Encoding)|Merge frequent subword pairs|GPT family|
|WordPiece|Similar to BPE, likelihood-based splits|BERT|
|SentencePiece|Language-agnostic BPE/unigram|T5, multilingual|

Key terms:

|Term|Meaning|
|---|---|
|Vocabulary size|Number of unique tokens (e.g. 50k)|
|Context length|Max tokens the model accepts|
|Special tokens|`[CLS]`, `[SEP]`, `<eos>`, `<pad>`|
|Padding|Fill short sequences to fixed length|
|Truncation|Cut long sequences to max length|

---

## 5. Embeddings

Map discrete tokens (or categories) to dense vectors.

|Type|Description|
|---|---|
|Word2Vec / GloVe|Static — one vector per word|
|Contextual (BERT, GPT)|Dynamic — vector depends on context|
|Learned embedding layer|`E ∈ ℝ^{V × d}` — trained end-to-end|
|Positional embeddings|Encode position in sequence|

## 6. Data Augmentation

Artificially expand training distribution to improve generalization.

### Images

|Technique|Note|
|---|---|
|Random crop / flip|Standard for CNNs|
|Color jitter|Brightness, contrast, saturation|
|Rotation / affine|Task-dependent|
|Mixup|`x̃ = λxᵢ + (1-λ)xⱼ` — blend two samples|
|CutMix|Replace patch of one image with another|
|RandAugment|Sample from a set of transforms randomly|

### Text

|Technique|Note|
|---|---|
|Back-translation|Translate to another language and back|
|Synonym replacement|Replace words with synonyms|
|Random deletion / swap|Low-level noise|
|Paraphrase with LLM|Strong augmentation for small datasets|

---

## 7. Data Pipeline

```
Raw data
  → Clean (nulls, duplicates, types)
  → Split (train/val/test)
  → Preprocess (fit on train only)
  → Augment (train only)
  → Batch + shuffle (train only)
  → Feed to model
```

|Concern|Note|
|---|---|
|Shuffle training data|Prevents order bias|
|Don't shuffle val/test|Reproducibility|
|Prefetch / parallel loading|Avoid GPU starvation|
|Reproducibility|Fix random seeds everywhere|

---

## 8. Common Data Mistakes

|Mistake|Why it's wrong|
|---|---|
|Fit scaler on full dataset|Leaks val/test stats into training|
|Augment val/test set|Evaluation should reflect real distribution|
|Ignore class imbalance|Model learns majority class; use oversampling, weights, or stratified split|
|Duplicate rows across splits|Inflates eval metrics|
|Wrong dtype|Strings treated as categories, ints as continuous, etc.|
