## Loss Functions

|Name|Formula|Use Case|
|---|---|---|
|MSE|`(1/n) Σ(y - ŷ)²`|Regression|
|MAE|`(1/n) Σ\|y - ŷ\|`|Regression (robust to outliers)|
|Cross-Entropy (binary)|`-[y log(ŷ) + (1-y) log(1-ŷ)]`|Binary classification|
|Cross-Entropy (categorical)|`-Σ y_i log(ŷ_i)`|Multi-class classification|
|KL Divergence|`Σ P(x) log(P(x)/Q(x))`|VAEs, distillation, PPO|
|Huber Loss|`½(y-ŷ)²` if `\|e\|≤δ`, else `δ\|e\| - ½δ²`|Regression (blend of MSE/MAE)|
|Hinge|`max(0, 1 - y·ŷ)`|SVMs|
|NLL|`-Σ log P(y_i)`|Language models|

## Optimization

### Gradient Descent Variants

|Variant|Update Rule|Note|
|---|---|---|
|SGD|`θ ← θ - η∇L`|One sample or mini-batch|
|Momentum|`v ← βv + ∇L`, `θ ← θ - ηv`|Dampens oscillations|
|RMSProp|`θ ← θ - η/(√v+ε) · ∇L`|Adaptive per-param LR|
|Adam|`θ ← θ - η · m̂/(√v̂+ε)`|Momentum + RMSProp|
|AdamW|Adam + weight decay on `θ` directly|Fixes L2 reg in Adam|

### Learning Rate Schedules

|Schedule|Behavior|
|---|---|
|Constant|Fixed `η` throughout|
|Step decay|Drop `η` by factor every N steps|
|Cosine annealing|`η` follows cosine curve to 0|
|Warmup + decay|Ramp up then decay (standard in transformers)|
|Cyclical|Oscillates between min/max `η`|

## Backpropagation

Core idea: chain rule applied backward through the compute graph.

```
∂L/∂w = ∂L/∂ŷ · ∂ŷ/∂w
```

Steps:

1. **Forward pass** — compute predictions and loss
2. **Backward pass** — compute gradients via chain rule
3. **Update** — step weights with optimizer

→ See `Math/Calculus/Chain_Rule_Backprop`

## Activation Functions

|Name|Formula|Use|
|---|---|---|
|ReLU|`max(0, x)`|Default hidden layers|
|Leaky ReLU|`max(αx, x)`|Fixes dead neurons|
|GELU|`x · Φ(x)`|Transformers|
|Sigmoid|`1/(1+e⁻ˣ)`|Binary output|
|Softmax|`eˣⁱ / Σeˣʲ`|Multi-class output|
|Tanh|`(eˣ - e⁻ˣ)/(eˣ + e⁻ˣ)`|RNNs, normalized output|

## Regularization

|Method|What It Does|Formula / Note|
|---|---|---|
|L2 (weight decay)|Penalizes large weights|`Loss + λΣw²`|
|L1|Penalizes non-zero weights (sparse)|`Loss + λΣ\|w\|`|
|Dropout|Randomly zeroes activations during training|Keep prob `p`|
|Batch Norm|Normalizes layer inputs per batch|`(x - μ)/σ · γ + β`|
|Layer Norm|Normalizes across features (not batch)|Same formula, different axis|
|Early Stopping|Halt when val loss stops improving|Not a penalty, a stopping rule|
|Data Augmentation|Artificially expand training distribution|Task-specific|

## Overfitting / Underfitting

|Problem|Symptom|Fix|
|---|---|---|
|Underfitting|High train loss + high val loss|Bigger model, more epochs, less regularization|
|Overfitting|Low train loss + high val loss|More data, dropout, L2, early stopping|
|Good fit|Train ≈ val loss, both low|—|

Bias-variance tradeoff: `Error = Bias² + Variance + Noise`

→ See `Math/Statistics/Bias_Variance`

## Initialization

|Method|Formula|Use|
|---|---|---|
|Xavier / Glorot|`U(-√(6/(n_in+n_out)), +√(...))`|Sigmoid / Tanh|
|He|`N(0, √(2/n_in))`|ReLU|
|Zeros|`w = 0`|Never for weights (symmetry breaking)|

Bad init → vanishing or exploding gradients.

## Batching

|Term|Meaning|
|---|---|
|Batch size|Samples per gradient update|
|Epoch|One full pass over the dataset|
|Iteration|One gradient update (one batch)|
|Mini-batch GD|Typical default — batch size 32–512|

Larger batch → more stable gradients, less noise, needs higher LR, more memory.
