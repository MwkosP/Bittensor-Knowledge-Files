



# ML — Training

---

## The Training Loop

```
for each epoch:
    for each batch:
        1. Forward pass  → compute ŷ and loss
        2. Backward pass → compute gradients (backprop)
        3. Optimizer step → update weights
        4. Zero gradients → reset for next batch
```

In PyTorch:

```python
optimizer.zero_grad()
loss = criterion(model(X), y)
loss.backward()
optimizer.step()
```

---

## Batching

| Term        | Meaning                         |
| ----------- | ------------------------------- |
| Batch size  | Samples per gradient update     |
| Epoch       | One full pass over dataset      |
| Iteration   | One gradient update (one batch) |
| Steps/epoch | `ceil(N / batch_size)`          |

|Batch size|Effect|
|---|---|
|Small (8–32)|Noisy gradients, regularizing, slow|
|Large (256–2048)|Stable gradients, needs LR scaling|
|Gradient accumulation|Simulate large batch on small GPU — accumulate N steps before update|

## Learning Rate

Most important hyperparameter. Too high → diverge. Too low → stall.

### Schedules

|Schedule|Behavior|Use|
|---|---|---|
|Constant|Fixed `η`|Simple baselines|
|Step decay|Drop by factor every N steps|CNNs|
|Cosine annealing|Smooth decay to 0|Most modern training|
|Linear warmup + decay|Ramp up, then decay|Transformers (standard)|
|Cyclical (CLR)|Oscillates min↔max|Helps escape local minima|

**LR scaling rule:** if you scale batch size by `k`, scale `η` by `√k` (or `k` for linear scaling).

---

## Gradient Issues

|Problem|Symptom|Fix|
|---|---|---|
|Exploding gradients|Loss → NaN, huge weight updates|Gradient clipping: `‖g‖ ≤ max_norm`|
|Vanishing gradients|Early layers don't learn|ReLU, residuals, good init, BatchNorm|
|Dead neurons|ReLU outputs stuck at 0|Leaky ReLU, lower LR, better init|

**Gradient clipping:**

```
g ← g · (max_norm / max(‖g‖, max_norm))
```

## Regularization During Training

|Method|When to use|
|---|---|
|Dropout|After linear/attention layers; off at inference|
|Weight decay (L2)|Almost always; built into AdamW|
|Early stopping|Monitor val loss; stop when it stops improving|
|Label smoothing|Classification; prevents overconfident softmax|
|Gradient clipping|Transformers, RNNs always|

**Label smoothing:**

```
y_smooth = (1 - ε)·y_hard + ε/K
```

## Mixed Precision Training

Use `float16` (or `bfloat16`) for forward/backward, `float32` for optimizer state.

|Type|Memory|Speed|Precision|
|---|---|---|---|
|FP32|High|Slow|Full|
|FP16|Half|Fast|Can underflow|
|BF16|Half|Fast|Better range than FP16|

Requires **loss scaling** with FP16 to avoid underflow.  
BF16 is preferred for LLMs (no loss scaling needed).

## Checkpointing

Save model state periodically to recover from crashes or pick best model.

What to save:

```
model.state_dict()
optimizer.state_dict()
scheduler.state_dict()
epoch / step number
best val loss
```

Strategies:

- Save every N steps
- Save on best val metric
- Keep last K checkpoints only

## Distributed Training

| Strategy            | How                                        | Use when                    |
| ------------------- | ------------------------------------------ | --------------------------- |
| Data Parallel (DDP) | Same model on each GPU, split batches      | Most common — fits on 1 GPU |
| Model Parallel      | Split model across GPUs                    | Model too large for 1 GPU   |
| Pipeline Parallel   | Split layers across GPUs in stages         | Very deep models            |
| ZeRO (DeepSpeed)    | Shard optimizer state / gradients / params | LLM training                |

## Hyperparameter Tuning

|Method|Idea|
|---|---|
|Grid search|Try all combinations|
|Random search|Sample randomly — usually better than grid|
|Bayesian optimization|Model the loss surface, sample intelligently|
|Learning rate finder|Sweep LR from low→high, pick before loss spikes|

## Common Training Recipes

### CNN (image classification)

```
Optimizer : SGD + momentum 0.9
LR        : 0.1, cosine decay
Batch     : 256
Regulariz.: weight decay 1e-4, dropout, augmentation
```

### Transformer / LLM

```
Optimizer : AdamW (β1=0.9, β2=0.95, ε=1e-8)
LR        : warmup 1–5% of steps, cosine decay
Batch     : large (accumulate if needed)
Precision : BF16
Regulariz.: weight decay 0.1, gradient clip 1.0
```
