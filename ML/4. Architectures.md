## 1. Classical ML

### Supervised — Classification

| Model                       | Core Idea                                       | Key Formula / Note                  |
| --------------------------- | ----------------------------------------------- | ----------------------------------- |
| Logistic Regression         | Linear model + sigmoid                          | `ŷ = σ(Wx + b)`                     |
| k-NN                        | Classify by majority of k nearest neighbors     | Distance metric (Euclidean, cosine) |
| Decision Tree               | Split features by information gain / Gini       | Greedy, axis-aligned splits         |
| Random Forest               | Ensemble of trees (bagging)                     | Majority vote over N trees          |
| Gradient Boosting (XGBoost) | Ensemble: each tree corrects previous residuals | `F_m(x) = F_{m-1}(x) + η·h_m(x)`    |
| SVM                         | Max-margin hyperplane                           | `min ½‖w‖² s.t. y_i(w·x_i+b) ≥ 1`   |
| Naive Bayes                 | Bayes theorem + feature independence assumption | `P(y\|x) ∝ P(y)ΠP(x_i\|y)`          |
| ...                         |                                                 |                                     |

### Supervised — Regression

| Model                 | Core Idea                                  | Key Formula / Note         |
| --------------------- | ------------------------------------------ | -------------------------- |
| Linear Regression     | Fit a line                                 | `ŷ = Wx + b`, minimize MSE |
| Ridge                 | Linear + L2 penalty                        | `‖y - Xw‖² + λ‖w‖²`        |
| Lasso                 | Linear + L1 penalty (sparse)               | `‖y - Xw‖² + λ‖w‖₁`        |
| Polynomial Regression | Linear on engineered features `x, x², x³…` | Still linear in weights    |
| ...                   |                                            |                            |

### Unsupervised

| Model   | Core Idea                                   | Note                             |
| ------- | ------------------------------------------- | -------------------------------- |
| k-Means | Assign points to nearest centroid, refit    | Sensitive to init; use k-means++ |
| PCA     | Project to top-k eigenvectors of covariance | Dimensionality reduction         |
| DBSCAN  | Cluster by density, marks outliers          | No need to specify k             |
| GMM     | Soft clusters modeled as Gaussians          | Uses EM algorithm                |
| ...     |                                             |                                  |

## 2. Neural Networks

### MLP (Multi-Layer Perceptron)

Fully connected layers stacked.

```
Input → [Linear → Activation] × N → Output
```

|Part|Detail|
|---|---|
|Layer|`y = σ(Wx + b)`|
|Activation|ReLU (hidden), Softmax/Sigmoid (output)|
|Loss|Cross-entropy (classif.), MSE (regression)|
|Use|Tabular data, simple baselines|

---

### CNN (Convolutional Neural Network)

Designed for spatially structured data (images, audio, time series).

|Operation|Formula / Note|
|---|---|
|Convolution|`(I * K)[i,j] = ΣΣ I[i+m, j+n]·K[m,n]`|
|Pooling|Max or average over a window (downsampling)|
|Padding|Keep spatial size: `same` padding|
|Output size|`(W - K + 2P)/S + 1`|

```
Input → [Conv → BN → ReLU → Pool] × N → Flatten → MLP → Output
```

|Hyperparameter|Meaning|
|---|---|
|Kernel size|Receptive field (e.g. 3×3)|
|Stride|Step size of convolution|
|Channels|Number of filters per layer|
|Depth|Number of conv blocks|

Use: image classification, object detection, segmentation.

---

### RNN (Recurrent Neural Network)

Processes sequences step-by-step, carries hidden state.

```
h_t = tanh(W_h · h_{t-1} + W_x · x_t + b)
ŷ_t = W_y · h_t
```

**Problem:** vanishing/exploding gradients over long sequences.

---

### LSTM (Long Short-Term Memory)

RNN with gating to control what to remember / forget.

|Gate|Formula|Purpose|
|---|---|---|
|Forget|`f_t = σ(W_f·[h_{t-1}, x_t])`|What to erase from cell|
|Input|`i_t = σ(W_i·[h_{t-1}, x_t])`|What new info to add|
|Output|`o_t = σ(W_o·[h_{t-1}, x_t])`|What to expose as h_t|
|Cell|`c_t = f_t⊙c_{t-1} + i_t⊙tanh(...)`|Long-term memory|

Use: time series, NLP before transformers, speech.

---

### GRU (Gated Recurrent Unit)

Simplified LSTM — 2 gates instead of 3, no separate cell state.  
Faster to train, comparable performance on many tasks.

## 3. Transformer (Classical)

The architecture that replaced RNNs for sequences.  
Core idea: **attention over all positions simultaneously**, no recurrence.

### Self-Attention

```
Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V
```

|Variable|Meaning|
|---|---|
|Q|Query — what am I looking for?|
|K|Key — what do I offer?|
|V|Value — what do I return?|
|d_k|Key dimension (scaling factor)|

### Multi-Head Attention

Run attention `h` times in parallel with different projections, concatenate.

```
MultiHead(Q,K,V) = Concat(head_1,...,head_h) · W_O
```

### Transformer Block

```
x → LayerNorm → Multi-Head Attention → Residual
  → LayerNorm → FFN (Linear → ReLU → Linear) → Residual
```

|Component|Purpose|
|---|---|
|Residual connections|Gradient flow, avoids vanishing|
|Layer norm|Stabilizes training|
|FFN|Per-position nonlinear transformation|
|Positional encoding|Inject sequence order (no recurrence)|

### Positional Encoding (sinusoidal)

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

---

### Encoder vs Decoder

|Type|What it does|Example Use|
|---|---|---|
|Encoder only|Produces contextual embeddings|BERT, classification|
|Decoder only|Generates tokens autoregressively|GPT, language modeling|
|Encoder-Decoder|Maps one sequence to another|T5, translation|

## 4. Classical LLM

A **decoder-only Transformer** trained to predict the next token.

```
Input tokens → Token Embeddings + Positional Encoding
             → N × Transformer Blocks (masked self-attention)
             → Linear → Softmax over vocabulary
             → Next token distribution
```

|Component|Detail|
|---|---|
|Tokenizer|Converts text ↔ integer IDs (BPE, WordPiece)|
|Embedding layer|`E ∈ ℝ^{V × d_model}` — lookup table|
|Causal masking|Each token attends only to past tokens|
|Training objective|NLL / cross-entropy on next token: `-log P(x_t \| x_{<t})`|
|Inference|Sample or argmax from output distribution|

### Key Sizes (rough intuition)

|Scale|Params|Layers|d_model|Heads|
|---|---|---|---|---|
|Small|~125M|12|768|12|
|Medium|~1B|24|2048|16|
|Large|~7B|32|4096|32|

## Architecture Decision Map

| Data type          | Start with                  |
| ------------------ | --------------------------- |
| Tabular            | XGBoost or MLP              |
| Image              | CNN                         |
| Sequence (short)   | LSTM / GRU                  |
| Sequence (long)    | Transformer                 |
| Text generation    | Decoder-only Transformer    |
| Text understanding | Encoder-only Transformer    |
| Seq-to-seq         | Encoder-Decoder Transformer |
| No labels          | k-Means, PCA, DBSCAN        |
