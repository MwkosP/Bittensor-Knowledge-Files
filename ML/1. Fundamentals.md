## Why Each Pillar Matters for ML

### Linear Algebra

Every piece of data in ML is a vector. Every model transformation is a matrix multiplication. Every embedding, every attention score, every weight update — linear algebra.

You cannot understand what a neural network layer _is_ without it. `y = Wx + b` is the entire forward pass of a linear layer, and that single equation is linear algebra.

→ Go here when you see: weight matrices, embeddings, PCA, attention, dot products, dimensionality

### Calculus

Training a model means finding weights that minimize a loss function. The only way to do that at scale is gradients — which direction do I move each weight to reduce loss? That is calculus.

Backpropagation is just the chain rule applied repeatedly. Every optimizer (SGD, Adam) is built on top of gradients. Without calculus, there is no learning.

→ Go here when you see: gradients, backpropagation, learning rate, optimizers, loss curves, second-order methods

### Statistics

A model trained on finite data needs to generalize to data it has never seen. Statistics is how you reason about that gap. Bias, variance, overfitting, underfitting, confidence — all statistics.

Also underpins everything in evaluation: every metric you compute on a test set is a statistical estimate.

→ Go here when you see: overfitting, regularization, train/val/test splits, confidence intervals, evaluation metrics, distributions

### Information Theory

The standard loss function for classification — cross-entropy — comes directly from information theory. KL divergence, which appears in VAEs, PPO, and knowledge distillation, is information theory. The Fisher information in Bittensor's pruning score is information theory.

It gives you the _why_ behind loss functions, not just the formula.

→ Go here when you see: cross-entropy loss, KL divergence, entropy of predictions, mutual information, Fisher information, VAE loss

## The One Paragraph That Connects Everything

A neural network takes input data (a vector — **Linear Algebra**), passes it through layers of matrix multiplications, and produces an output distribution. During training, we measure how wrong that output is using cross-entropy loss (negative log-likelihood — **Information Theory**). We then compute how each weight contributed to that error by applying the chain rule backward through the network (**Calculus**). We update weights by stepping in the direction that reduces loss. We repeat this on random batches of data, hoping the model learns patterns that generalize beyond the training set (**Statistics**).

That is the entirety of deep learning. Everything else is a variation on this loop.
