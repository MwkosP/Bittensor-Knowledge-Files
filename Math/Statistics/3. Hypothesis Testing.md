


# Statistics — Hypothesis Testing

---

## Core Idea

You have a claim about the world. You collect data. Does the data support or contradict the claim?

|Term|Meaning|
|---|---|
|$H_0$ (null hypothesis)|Default assumption — "nothing interesting is happening"|
|$H_1$ (alternative hypothesis)|What you're trying to show|
|Test statistic|A number computed from data that summarizes evidence|
|p-value|Probability of seeing data this extreme **if $H_0$ were true**|
|$\alpha$|Significance threshold — typically 0.05|

---

## The Testing Procedure

$$ \begin{aligned} 1.&\quad \text{State } H_0 \text{ and } H_1 \\ 2.&\quad \text{Choose significance level } \alpha \\ 3.&\quad \text{Compute test statistic from data} \\ 4.&\quad \text{Compute p-value} \\ 5.&\quad \text{If p-value} < \alpha \text{: reject } H_0 \end{aligned} $$

**Rejecting $H_0$ does not prove $H_1$** — it only means the data is unlikely under $H_0$.

---

## p-value

$$ p = P(\text{data this extreme} \mid H_0 \text{ is true}) $$

|p-value|Interpretation|
|---|---|
|$p < 0.05$|Statistically significant — reject $H_0$|
|$p < 0.01$|Strongly significant|
|$p \geq 0.05$|Fail to reject $H_0$ — insufficient evidence|

**Common misconception:** p-value is NOT the probability that $H_0$ is true.

---

## Type I and Type II Errors

|Decision|$H_0$ True|$H_0$ False|
|---|---|---|
|Reject $H_0$|**Type I error** (false positive) — rate $= \alpha$|Correct (true positive)|
|Fail to reject $H_0$|Correct (true negative)|**Type II error** (false negative) — rate $= \beta$|

$$ \text{Power} = 1 - \beta = P(\text{reject } H_0 \mid H_0 \text{ is false}) $$

Higher power = better at detecting real effects.

---

## Common Tests

### z-test

Population std known, large sample.

$$ z = \frac{\bar{X} - \mu_0}{\sigma / \sqrt{n}} $$

### t-test

Population std unknown, small sample.

$$ t = \frac{\bar{X} - \mu_0}{s / \sqrt{n}}, \quad s = \hat{\sigma} $$

### Two-sample t-test

Compare means of two groups:

$$ t = \frac{\bar{X}_1 - \bar{X}_2}{\sqrt{\frac{s_1^2}{n_1} + \frac{s_2^2}{n_2}}} $$

**In ML:** compare model A accuracy vs model B accuracy on the same test set.

### Chi-squared test

Test if categorical frequencies match expected:

$$ \chi^2 = \sum_i \frac{(O_i - E_i)^2}{E_i} $$

---

## Confidence Intervals vs Hypothesis Tests

They are two sides of the same coin:

$$ \text{Reject } H_0 \text{ at level } \alpha \iff \theta_0 \notin \text{CI at level } (1-\alpha) $$

A 95% CI contains all values of $\theta_0$ that would **not** be rejected at $\alpha = 0.05$.

---

## Multiple Testing Problem

Running $k$ tests at $\alpha = 0.05$ → expected false positives $= 0.05k$.

**Bonferroni correction:** adjust threshold per test:

$$ \alpha_{\text{adjusted}} = \frac{\alpha}{k} $$

**In ML:** evaluating many metrics or hyperparameter combinations inflates false discovery rate.

---

## Permutation Test

Non-parametric — no distributional assumptions needed.

$$ \begin{aligned} 1.&\quad \text{Compute observed test statistic } T_{\text{obs}} \ 2.&\quad \text{Randomly shuffle labels many times} \ 3.&\quad \text{Compute } T \text{ on each shuffle} \ 4.&\quad p = \frac{\text{number of shuffles where } T \geq T_{\text{obs}}}{\text{total shuffles}} \end{aligned} $$

**In ML:** testing whether model A is significantly better than model B without assuming normality.

---

## McNemar's Test

Compare two classifiers on the **same** test set.

$$ \chi^2 = \frac{(b - c)^2}{b + c} $$

Where $b$ = samples A gets right, B gets wrong; $c$ = samples B gets right, A gets wrong.  
More appropriate than a two-sample t-test for comparing classifiers.

---

## Statistical vs Practical Significance

|Type|Question|
|---|---|
|Statistical significance|Is the effect real (not noise)?|
|Practical significance|Is the effect large enough to matter?|

A model improvement of 0.01% accuracy can be statistically significant on a large test set but practically meaningless. Always report **effect size**, not just p-value.
